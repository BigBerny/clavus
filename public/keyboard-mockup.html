Clavus Keyboard Extension — Feature Spec

Overview

A custom iOS keyboard extension that reads dictation transcripts from App Group shared storage and auto-inserts them into the active text field. The keyboard is a transparent glass overlay with a single row of format icons. Pre-select the channel, dictate via Action Button, text is processed and inserted — then the keyboard dismisses itself.

Interaction Flow

1.User opens Clavus keyboard. Format icons shown with auto-detected channel pre-selected. Keyboard area is transparent glass.
2.User can optionally change the channel or toggle EN before dictating.
3.User presses Action Button → Shortcut records + transcribes → writes to shared storage.
4.Keyboard detects the new transcript (0.5s polling). Transcript preview shown briefly above the icons.
5.Auto-insert begins immediately:
 • Insert / URL: text inserted directly. No LLM call.
 • Channel (Slack/Message/Email/Prompt): loading spinner appears as a ring around the active icon. Hermes reformulates, then inserts.
6.Text inserted → keyboard dismisses itself.
7.Override: If user taps a different icon before auto-insert completes, it cancels and restarts with the new channel.

Real-Time Polling

  • 0.5s Timer polls SharedStorage.activeEntries() while keyboard is visible.
  • New transcript ID detected → show preview, start auto-insert pipeline.
  • Timer starts in viewDidAppear, stops in viewDidDisappear.

Format Buttons

Icon-only glass circles, matching the macOS dictation widget. 38pt, frosted glass background, colored ring when active. Loading state is a spinner ring around the button — no separate loading bar.

Insert (icon) Slack (icon) Message (icon) Email (icon) Prompt (icon) | EN (toggle)

Conditional: URL text button appears when transcript looks like a URL. No other text labels.

Keyboard Transparency

  • Keyboard background is fully transparent with only backdrop-filter: blur(24px) saturate(200%) — the host app content shows through as a frosted blur.
  • On iOS 26: use .regularMaterial as the root background for system-level glass integration. On older iOS: .ultraThinMaterial.
  • No opaque background color. The buttons themselves have a subtle glass fill (oklch(0.15 / 0.4)) so they're visible against any content.

Auto-Dismiss

  • After text is inserted via textDocumentProxy.insertText(), call dismissKeyboard() after a ~0.3s delay.
  • No success flash or confirmation — the inserted text in the field is confirmation enough.

Height Budget

  • Idle: ~50px — format row + globe toolbar + home indicator.
  • With transcript: ~90px — preview + format row + globe + home indicator. Temporary — dismisses after insert.

SwiftUI Notes

  • Root surface: .background(.regularMaterial) on iOS 26, .ultraThinMaterial fallback. No ClavusTheme.bg fill.
  • Icons: 38pt glass circles with clavusGlassChip(cornerRadius: 19). Loading ring: Circle().trim(from: 0, to: 0.7).stroke(lineWidth: 2).rotationEffect(...) animated.
  • Auto-dismiss: textDocumentProxy.insertText(result) then DispatchQueue.main.asyncAfter(deadline: .now() + 0.3) { self.dismissKeyboard() }.
  • SF Symbols: doc.plaintext (Insert), number (Slack), bubble.left.fill (Message), envelope.fill (Email), bolt.fill (Prompt), link (URL).
